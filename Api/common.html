# API规范

# 字段意义统一

字段
code 1 = 成功 0 = 失败 -1 = 未登录
msg 返回的信息

# 返回的数据

像推广二维码，轮播图等，相对路径会拿不到图片，拼接后期维护又麻烦，最好在暂时写死成绝对路径，可以返回的是一张图片暂时写死

msg 返回的信息尽量全面，比如：手机号码不能为空！， 而不是“失败”

在联调时有数据的接口，返回数据。（方便检错）联调后不返回数据的接口返回空数组/空对象（不会报错can’t ready）

# 接口的分类参考

普通项目建议以首页，个人中心等进行接口分类，公用接口放在公共分类组下

# 细分请求方式

GET,POST 等

接口文档不要带域名（前缀）
